# Docs

Plans and design specs. Separate from `memory/` because these are *durable artifacts* of design thinking, not operator state.

## Folders

- **`plans/`** — Step-by-step implementation plans. Output of the `planning` skill (or `superpowers:writing-plans` if you use that plugin).
- **`specs/`** — Design specs and brainstorm outputs. Captures the WHY and the design space before you write a plan.

Naming convention: `YYYY-MM-DD-slug.md` so plans and specs sort chronologically.
